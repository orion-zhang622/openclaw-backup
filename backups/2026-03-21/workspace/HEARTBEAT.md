# HEARTBEAT.md - 定期检查清单

每次心跳触发时执行以下检查。

## 检查周期（轮换）

每 4 小时一次，但不同检查按优先级轮换，避免每次都检查全部。

## 每次心跳必做（基础状态汇报）

### 1. 系统状态快照
执行以下命令获取系统状态：
```bash
df -h /
free -h
uptime
ps aux --sort=-%mem | head -6
docker system df 2>/dev/null || echo "Docker not available"
du -sh ~/.openclaw 2>/dev/null || echo "N/A"
```

**阈值判断：**
- 磁盘 > 80% → 🔴 红色警告
- 内存 > 75% → 🟡 黄色提醒
- 负载 > 4 → 🟡 黄色提醒

### 2. OpenClaw 上下文状态
- 运行 `session_status` 获取上下文使用率
- 上下文 > 60% → 🟡 提醒
- 上下文 > 80% → 🔴 警告

### 3. 日历检查
- 接下来 4 小时内是否有会议？
- 如果有，提前提醒用户

### 4. 待办检查
- 使用 `memory_recall` 查询未完成任务（关键词：待办、任务、未完成）
- 有即将到期的事项吗？

### 5. 小红书自动化运营 ⭐ 每次心跳检查
**检查逻辑**:
1. 读取「自动化运营」多维表格，查看今日已执行的操作
2. 判断今日是否已完成养号目标:
   - 浏览: 15-20分钟
   - 点赞: 5-8篇
   - 收藏: 1-3篇
3. **如果未完成目标** → 立即执行剩余操作
4. **如果已完成目标** → 记录"今日已完成"，跳过
5. **无论是否执行** → 汇报当前进度

**执行策略**（根据阶段）:
- **养号期(Day 1-7)**: 浏览+点赞+收藏，不发内容
- **测试期(Day 8-14)**: 浏览+点赞+每天1篇
- **稳定期(Day 15+)**: 浏览+点赞+每天1-2篇

**发布审核**: 所有内容发布前必须发给你审核确认

**汇报格式**:
```
📕 小红书运营 [Day X]
├─ 今日进度: 浏览X分钟 | 点赞X篇 | 收藏X篇
├─ 目标完成: X%
├─ 本次执行: [操作列表]
└─ 状态: 已完成 / 待下次补足
```

| 指标 | 数值 | 状态 |
|------|------|------|
| 磁盘 | XG / YG (Z%) | 🟢🟡🔴 |
| 内存 | XG / YG (Z%) | 🟢🟡🔴 |
| 负载 | X.XX | 🟢🟡🔴 |

📅 日历：接下来4小时无会议 / 有N个会议
📝 待办：N个未完成任务
📬 消息：N条未读（如检查）

## 执行规则

1. **发现问题** → 发送详细提醒给用户
2. **一切正常** → 回复 HEARTBEAT_OK（不打扰）
3. **深夜时段**（23:00-08:00）→ 只检查紧急情况，普通问题延后到早上
